# Shokker Paint Booth — Tutorial Series

Welcome to the official SPB tutorial track. This is a hands-on, screen-by-screen walkthrough of every major workflow in Shokker Paint Booth, written for new users who want to be productive fast and for experienced users who want to make sure they aren't missing the good shortcuts.

If you can install software and use a mouse, you can finish this whole series in an afternoon and walk away painting iRacing-ready liveries.

---

Ten tutorials, each focused on one slice of the app. Each one is short enough to do over a coffee break and self-contained enough that you can drop in mid-series if you only care about a specific feature.

## Recommended order

Tutorial 1 first, no exceptions. It gives you the whole pipeline in ten minutes so the rest of the series has context to attach to. After Tutorial 1, the rest are written so you can take them in any order, but they do reference each other and the **strongly recommended path** is just 1 through 10 in order.

The one warning: **don't skip Tutorial 3 (Zones Explained).** Zones are the single most important concept in SPB. Almost every confused-newcomer question on the Discord traces back to misunderstanding zones. Fifteen minutes there saves you hours of head-scratching later.

## Prerequisite knowledge

Almost none. You should be comfortable with:

- Installing and launching desktop apps on Windows
- Basic file management (folders, finding `Documents\iRacing\paint`)
- The general idea that an "image layer" is a thing (Photoshop background helps but isn't required)

You do **not** need to know:

- Photoshop
- Photogrammetry, UV mapping, or 3D
- Color theory, PBR shading, or any technical art jargon
- iRacing's painting system (we explain what we need)

If you've ever opened MS Paint and made a face that looks vaguely like a duck, you have the prerequisite art skills.

## How to follow along

1. **Install SPB.** See [SPB_QUICKSTART.md](../SPB_QUICKSTART.md) for the install path.
2. **Open SPB.** The first launch loads the Chevy Silverado 2019 demo PSD automatically. That's our shared playground for the whole series.
3. **Read a tutorial alongside the running app.** Side-by-side windows beat alt-tabbing.
4. **Do the "Try it yourself" challenge at the end of each one.** This is where the learning actually sticks. Reading is theory; clicking is muscle memory.
5. **If something breaks, hit the Troubleshooting section** at the end of each tutorial before bouncing to Discord.

Most tutorials assume you have iRacing installed and the Live Link path configured (see Tutorial 1). If you don't have iRacing, you can still complete every tutorial — just skip the "see it in iRacing" steps. The PNG and TGA exports work standalone.

## Where to ask questions

Three places, in order of speed:

1. **The in-app `?` overlay.** Press `?` at any time to see every keyboard shortcut for the panel you're in.
2. **[SPB_FAQ.md](../SPB_FAQ.md).** Forty of the most common questions, answered in plain English.
3. **The Shokker Discord.** Look for the `#paint-booth-help` channel. Real humans, real fast. The QR code is in the app's About screen.

For long-form discussions, design crit, and "look at this thing I made" — the `#livery-showcase` channel on Discord is the place. We love seeing what people build.

## Contributing additional tutorials

If you discover a workflow worth teaching, write it up. The bar is low and the audience is hungry.

**Format guidelines:**

- 800 to 1200 words is the sweet spot. Long enough to be useful, short enough to finish in a sitting.
- Open with a "What you'll learn" bullet list.
- Use numbered steps. Each step gets one action.
- Drop screenshot placeholders inline: `![Step 4](docs/img/your-tut-step4.png)`. We'll fill in the images.
- End with a "Try it yourself" challenge, a "Troubleshooting" mini-FAQ, and a "Next up" pointer.
- Friendly, conversational tone. Pretend you're showing a friend.

Submit a PR against the `tutorials/` folder. Maintainers review weekly. If accepted, your tutorial gets added to the index here and your name in `AUTHORS.md`.
