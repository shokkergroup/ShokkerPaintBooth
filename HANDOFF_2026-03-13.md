## 🎯 THE ORIGINAL GOAL THAT STARTED ALL OF THIS

The user wanted finishes (Dark Matter, Holographic, Coagulated Blood, Foundation Primer, etc.) to behave like this:

- **At 100% Base Strength:** Full finish effect is applied ON TOP of the user's base color. The color should still show through — Dark Matter makes it dark/cloudy, Holographic adds rainbow iridescence, Coagulated Blood adds the texture — but the underlying BLUE (or pink, or red) car should still be recognizable
- **At 50% Base Strength:** Halfway between the full finish effect and the pure base color
- **At 0% Base Strength:** The car returns to exactly the original base color, no finish

**The CORRECT reference is Coagulated Blood (Candy & Pearl category)** — at 100% it preserves blue with the coagulated finish on top. Foundation Primer nearly works right too, giving a grey tint but preserving the blue undertone.

**The PROBLEM** is bases like Dark Matter, Black Hole, Holographic don't work this way. Their `paint_fn` scripts hardcode color crushing at the very first line (e.g., `core = np.clip(paint * 0.05, 0, 1)`) which ignores `pm` entirely and forces the output toward absolute black regardless of slider position.

**The Hard Light approach FAILED** because:
1. Dark Matter's script crushes to ~5% brightness before the blend math runs
2. Holographic's rainbow stripes ARE the finish; stripping them to 0 removes the entire look
3. 1100+ specials would all need individual audit/patching — not scalable

## 🔧 THE REAL PROBLEM THAT STILL NEEDS SOLVING

**The right approach** (that we did NOT get to implement) is:

Each base `paint_fn` in `spec_paint.py` needs to respect the `pm` parameter faithfully. When `pm = 0.0`, the function should return the paint buffer UNCHANGED. When `pm = 1.0`, it applies its full effect.

The issue is that many heavy finishes (Dark Matter, Black Hole, Plasma Core, Holographic, etc.) have lines like:
```python
paint[:,:,0] = np.clip(paint[:,:,0] * 0.05 + ..., 0, 1)  # ignores pm
```

These should be:
```python
effect = np.clip(paint[:,:,0] * 0.05 + ..., 0, 1)
paint[:,:,0] = paint[:,:,0] * (1.0 - pm) + effect * pm  # respects pm
```

This is the fundamental fix — a **linear interpolation between the original color and the finish effect, scaled by `pm`**. At `pm=0` you get pure base color. At `pm=1` you get pure finish. At `pm=0.5` you get half and half.

**The scope:** This needs to be done for the "heavy" finishes only — the ones that currently crush or replace the base color. Most light finishes (Candy, Pearl, some Metalbacks) already work correctly. The primary offenders are in the "Extreme & Experimental", "Exotic Metal", "Industrial & Tactical" categories.

**Suggested approach for Cursor:**
1. Search `spec_paint.py` for `def paint_` functions
2. For each one, check if it does `paint[:,:,N] = np.clip(paint[:,:,N] * <very small number>...` — that's a crusher
3. Wrap the crushing logic with the pm interpolation pattern above
4. Do NOT touch `compose.py` — it's stable now
5. Do NOT touch `base_strength` parameter flow in `compose.py` — it's correct
6. Test with Dark Matter on a blue car: at 0% should be blue, at 100% should be dark/cloudy but still recognizably blue (not pitch black)
